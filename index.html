👾 Git 小怪獸冒險之旅 👾

跟著可愛的小怪獸一起學習 Git 版本控制的七大指令！

➕

`git add` - 準備行李

想像你在打包行李。`git add .` 就是把房間裡所有要帶走的衣物 (變更的檔案) 放到你的行李箱 (暫存區) 裡。

git add .
📦

`git commit` - 蓋章寄出

行李箱打包好後，貼上標籤並蓋章確認 (提交變更)，告訴 Git 這次的變動是什麼意思，形成一個完整的歷史記錄。

git commit -m "新增了首頁設計"
🚀

`git push` - 發射火箭

你的行李 (提交) 已經準備好了，`git push` 就像發射火箭，將你的本地變更發送到遠端伺服器 (如 GitHub)。

git push origin main
🎣

`git pull` - 釣魚更新

你想看看遠端伺服器有沒有別人新增的好東西。`git pull` 就像釣魚，把遠端的最新程式碼抓下來並合併到你的本地專案。

git pull origin main
⏪

`git revert` - 時光倒流

發現最新的行李標籤寫錯了！`git revert` 不會刪除歷史，而是建立一個新的「反向提交」，撤銷之前的特定變更，是比較安全的做法。

git revert [commit_hash]
🚪

`git checkout` - 任意門

想要回到過去的某個時間點，或是切換到另一個平行世界 (分支) 去看看？`git checkout` 就是那扇神奇的任意門。

git checkout dev-branch
git checkout [commit_hash]
🤝

`git merge` - 合體進化

當你在平行世界 (分支) 完成任務後，`git merge` 可以把那個世界的成果帶回主線，讓專案合體進化！

git merge feature-branch

學習愉快！祝您成為 Git 大師！🎉
